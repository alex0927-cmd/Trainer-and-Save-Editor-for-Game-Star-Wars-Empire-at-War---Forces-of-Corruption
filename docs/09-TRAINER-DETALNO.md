## Призначення

`EAW_FOC_Trainer` — трейнер реального часу для процесу `swfoc.exe` (Star Wars: Empire at War — Forces of Corruption).

| Властивість | Значення |
|-------------|----------|
| Мова інтерфейсу | Українська |
| Ціль | Чити **лише для гравця**, не для ШІ |
| Зміни на диску | **Немає** (працює в RAM) |
| Потрібен перезапуск гри | Ні (крім XML-модів окремо) |

**Вихідний код:**

| Файл | Роль |
|------|------|
| `trainer/app.py` | GUI (tkinter) |
| `trainer/cheats.py` | Логіка 11 читів, auto-bind, freeze |
| `trainer/scanner.py` | Сканер пам'яті, AoB, snapshot diff |
| `trainer/hotkeys.py` | Глобальні гарячі клавіші |
| `trainer/hotkey_store.py` | Збереження hotkey у JSON |
| `trainer/file_logger.py` | Запис журналу у файл |
| `trainer/calibration_store.py` | Збереження адрес RAM у JSON |
| `trainer_main.py` | Точка входу для exe / bat |
| `shared/process.py` | Підключення до процесу гри |

## Вимоги та запуск

### Системні вимоги

| Вимога | Деталі |
|--------|--------|
| OS | Windows 10 / 11 |
| Права | **Адміністратор** (OpenProcess, глобальні hotkey) |
| Гра | Запущена `swfoc.exe` |
| Python (для .bat) | 3.10+ з `pymem`, `keyboard` |
| Антивірус | Можливий false positive на exe |

### Способи запуску

#### Спосіб 1 — рекомендований: `Запуск_Трейнера.bat`

```
EAW_FOC_Tools\Запуск_Трейнера.bat
```

Що робить bat-файл:

1. Запитує права адміністратора (UAC)
2. Встановлює `PYTHONPATH` на корінь `EAW_FOC_Tools`
3. Перевіряє наявність `pymem` та `keyboard`, встановлює при потребі
4. Запускає `trainer_main.py` через `pythonw` (без консольного вікна)

**Чому bat, а не exe:** Windows Security / SmartScreen часто блокує непідписаний `EAW_FOC_Trainer.exe`. Запуск через Python обходить це.

#### Спосіб 2 — готовий exe

```
EAW_FOC_Tools\dist\EAW_FOC_Trainer.exe
```

ПКМ → **Запуск від імені адміністратора**. Якщо заблоковано — додайте виняток у Defender або використовуйте bat.

#### Спосіб 3 — розробка

```bash
cd EAW_FOC_Tools
set PYTHONPATH=.
python trainer_main.py
```

### Типовий сценарій сесії

```
1. Запустити swfoc.exe → завантажити сейв / скірміш / кампанію
2. Запустити трейнер від адміна
3. «Підключитися»
4. Авто-зв'язати кредити (2 кроки)
5. Ввести бажані значення → «Застосувати»
6. Увімкнути потрібні чити
7. (Опційно) призначити гарячі клавіші
8. (Опційно) увімкнути запис у файл для діагностики
```

## Інтерфейс (повний опис)

Вікно: **780×820** пікселів, темна тема (`#1a1a2e`).

### Верхня панель — підключення

| Елемент | Дія |
|---------|-----|
| **Процес** (combobox) | Список `swfoc.exe (PID …)` / `sweaw.exe` |
| **Оновити** | Пересканувати запущені процеси |
| **Підключитися** | `attach()` → створити `PlayerTrainer` → запустити `_maintenance_loop` |
| **Статус** | Зелений = підключено, червоний = ні |

Після підключення автоматично:

- зчитуються live-значення з RAM (якщо є збережені зв'язування)
- запускається таймер оновлення «Поточне» кожні **1 секунду**
- активуються глобальні гарячі клавіші з `trainer_hotkeys.json`

### Блок «Значення з гри»

Чотири рядки параметрів:

| Ключ | Назва в UI | Тип у RAM |
|------|------------|-----------|
| `credits` | Галактичні кредити | int / float |
| `tactical_credits` | Тактичні кредити | int / float |
| `health` | HP юніта | float |
| `cooldown` | Перезарядка (сек) | float |

Кожен рядок містить:

| Колонка / кнопка | Опис |
|------------------|------|
| **Поточне** | Зелене число з пам'яті гри; оновлюється кожну 1 с |
| **Бажане** | Поле вводу — ваше цільове значення |
| **Зчитати** | Копіює «Поточне» з гри в «Бажане» |
| **⌨** (біля Зчитати) | Призначити гарячу клавішу на «Зчитати» |
| **Застосувати** | Записує «Бажане» у RAM гри |
| **⌨** (біля Застосувати) | Призначити гарячу клавішу на «Застосувати» |
| **Авто-зв'язати** | 2-крокове знаходження адреси в RAM |

Кнопка **«Оновити все з гри»** — зчитує всі 4 параметри одразу.

### Блок «Чити (+10)»

11 чекбоксів — див. [таблицю читів](#список-читів-10).

Кожен рядок має:

- чекбокс увімк/вимк
- підказку сірим текстом
- **⌨** — призначити гарячу клавішу на **перемикання** чита

## Робота зі значеннями з гри

### Концепція «зв'язування» (binding)

Щоб записати кредити **саме вашого** гравця (а не ШІ), трейнер повинен знати **адреси в RAM**, де зберігається ваш рахунок. Це називається **зв'язування**.

Два методи:

| Метод | Коли використовувати |
|-------|---------------------|
| **Авто-зв'язати** (рекомендовано) | Не потрібно вводити точне число вручну |
| Класичне сканування (API у `cheats.py`) | Програмний доступ через `calibrate_credits` / `refine_credits` |

Детально про алгоритм: [16-AVTO-ZVYAZUVANNYA-TA-ZNACHENNYA.md](16-AVTO-ZVYAZUVANNYA-TA-ZNACHENNYA.md).

### Авто-зв'язати — покроково (кредити)

```
Крок 1: Натиснути «Авто-зв'язати»
        → трейнер зберігає snapshot усіх int у діапазоні 0…99 999 999
        → у журналі: «Крок 1/2: запам'ятано N адрес»

Крок 2: У ГРІ змінити кредити (купити щось, отримати дохід)
        → знову «Авто-зв'язати»
        → порівняння snapshot → знайдені адреси, де значення змінилось
        → поля «Поточне» і «Бажане» заповнюються автоматично
```

### Застосувати власне значення

```
1. У «Бажане» ввести, наприклад, 9999999
2. «Застосувати»
3. Увімкнути чит «Безлімітні кредити» — maintenance loop триматиме це значення
```

`target_value` у `CheatState` зберігає ваше число; loop кожні 150 мс перезаписує його в RAM.

### Зчитати з гри

Кнопка **«Зчитати»** викликає `pull_current_into_target()` — читає RAM і копіює в поле «Бажане».

### Live-оновлення

Кожну **1 секунду** `_update_live_labels()` читає `get_live_values()` і оновлює зелені мітки «Поточне».

## Список читів (+10)

| ID | Назва в UI | Механізм | Player-only як |
|----|------------|----------|----------------|
| `credits` | 1. Безлімітні кредити (галактика) | freeze RAM кожні 150 мс | зв'язані **ваші** адреси |
| `tactical_credits` | 2. Безлімітні тактичні ресурси | freeze RAM | тактичний гаманець гравця |
| `god_mode` | 3. Безсмертя юнітів | AoB patch **або** HP freeze | HP вашого юніта / патерн власника |
| `no_cooldown` | 4. Здібності без перезарядки | AoB NOP **або** freeze float | таймер **вашої** здібності |
| `unit_cap` | 5. Нескінченний ліміт юнітів | патч лічильника pop | патерн GC cap |
| `instant_build` | 6. Миттєве будівництво | патч швидкості build | хук виробництва гравця |
| `free_build` | 7. Будівництво за 1 кредит | патч вартості | хук покупки гравця |
| `map_reveal` | 8. Відкрита карта (Maphack) | byte patch fog | лише відображення |
| `fast_speed` | 9. Швидкість гри x3 | патч множника часу | глобально (не дає ШІ грошей) |
| `weak_enemies` | 10. Слабкі вороги (x5 урон) | патч множника урону | вороги отримують більше |
| `super_damage` | + Бонус: ваш урон x10 | патч damage deal | ваші атаки |

### Залежності між читами та калібруванням

| Чит | Потрібне зв'язування? |
|-----|----------------------|
| `credits` | Так — авто-зв'язати або скан |
| `tactical_credits` | Так |
| `god_mode` | Бажано HP; AoB — опційно |
| `no_cooldown` | Бажано cooldown; AoB — опційно |
| `unit_cap` … `super_damage` | Ні (лише AoB у `swfoc.exe`) |

### Значення freeze за замовчуванням

```python
CREDIT_TARGET = 9_999_999.0
TACTICAL_CREDIT_TARGET = 999_999.0
```

Після «Застосувати» користувацьке значення **перезаписує** ці константи.

Запис у RAM: і `write_float`, і `write_int` — гра може читати обидва типи.

## Фоновий потік та freeze

`PlayerTrainer.start_loop()` запускає daemon-thread `_maintenance_loop`:

```python
while not self._stop.is_set():
    if self.credits.enabled:
        target = self.credits.target_value or self.CREDIT_TARGET
        for addr in self.credits.addresses:
            pm.write_float(addr, target)
            pm.write_int(addr, int(target))
    # ... tactical, god_mode HP restore, cooldown zero ...
    time.sleep(0.15)
```

**Частота:** 150 мс (~6.7 разів/сек).

**При закритті** `shutdown()`:

1. Зупиняє потік
2. Відновлює всі code patches з `patch_backup`
3. Закриває file logger (якщо був увімкнений)

## Збережені налаштування

| Файл | Що зберігає | Коли створюється |
|------|-------------|------------------|
| `trainer_calibration.json` | Адреси RAM (credits, tactical, health, cooldown) | Після авто-зв'язування |

## Усунення проблем

| Симптом | Причина | Рішення |
|---------|---------|---------|
| «Помилка підключення» | Немає прав адміна | Запуск від адміна |
| «Поточне» = — | Немає зв'язування | Авто-зв'язати (2 кроки) |
| Чит не вмикається | AoB не знайдено | Оновити гру / CE-таблиця для вашої версії exe |
| exe не запускається | Windows Security | `Запуск_Трейнера.bat` |
| `'bool' object is not callable` | Стара версія | `pm.is_64_bit` — властивість, не метод |
| Гаряча клавіша не працює в грі | Немає адміна | Bat від адміна |
| Кредити ШІ теж ростуть | XML-мод + freeze | Вимкніть XML-моди або використовуйте лише трейнер |

### Відомі виправлені баги (історія)

| Проблема | Виправлення |
|----------|-------------|
| `pm.is_64_bit()` | `pm.is_64_bit` (властивість pymem) |
| `ModuleNotFoundError: shared` | `trainer_main.py` + `PYTHONPATH` |
| Слабкий сканер | Сканування всієї readable RAM, не лише модулів |
| CE-патерни в pymem | `ce_pattern_to_bytes()` |
