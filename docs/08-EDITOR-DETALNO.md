# 08. EAW_FOC_Editor — детальний опис

## Призначення

`EAW_FOC_Editor.exe` — графічний інструмент для редагування **розпакованих XML** гри без ручного відкривання Notepad.

**Шлях:** `EAW_FOC_Tools\dist\EAW_FOC_Editor.exe`  
**Вихідний код:** `editor\app.py`  
**GUI-фреймворк:** tkinter (стандартна бібліотека Python)

---

## Інтерфейс

### Верхня панель

- Заголовок проєкту
- Попередження: для читів **тільки для гравця** використовуйте Trainer
- Поле **«Папка гри»** — шлях до `data2` (де `swfoc.exe`)
- Кнопка `...` — вибір папки через діалог Windows

### Вкладки (Notebook)

| Вкладка | Поля | Цільовий файл |
|---------|------|---------------|
| **Економіка** | ліміт кредитів, скірміш min/max, дохід планет | `GAMECONSTANTS.XML`, `PLANETS.XML` |
| **Бойові** | HP космос/суша, респавн героя, щити | `GAMECONSTANTS.XML` |
| **Виробництво** | `Production_Speed_Factor` | `GAMECONSTANTS.XML` |
| **Фракції** | множник кредитів, утримання флоту | `FACTIONS.XML`, `EXPANSION_FACTIONS.XML` |
| **Планети** | масова зміна `Planet_Credit_Value` | `PLANETS.XML` |

### Список XML-файлів

Показує всі `*.XML` / `*.xml` у `data2\Data\XML\` — для огляду, що саме перевизначає MEG.

### Кнопки

| Кнопка | Дія |
|--------|-----|
| **Завантажити значення** | читає поточні теги з диска в поля форми |
| **Зберегти зміни** | backup + запис XML |
| **Чит-пресет (макс.)** | заповнює поля максимальними значеннями (не зберігає автоматично!) |
| **Відновити ваніль** | заповнює поля типовими ванільними значеннями FOC |

### Журнал

Текстове поле внизу — статус операцій, помилки regex.

## Алгоритм збереження

```python
def _save(self):
    gc = xml_path / "GAMECONSTANTS.XML"
    backup_file(gc)                    # 1. резервна копія
    write_tag(gc, tag, value)          # 2. точечна заміна тегів

    factions = xml_path / "FACTIONS.XML"
    backup_file(factions)
    replace_all_tags(factions, "Credits_Accumulation_Factor", value)  # 3. всі фракції

    expansion = xml_path / "EXPANSION_FACTIONS.XML"
  # аналогічно

    planets = xml_path / "PLANETS.XML"
    set_planet_credits(planets, 99999)  # 4. regex всіх планет
```

### Формат backup

```
GAMECONSTANTS.XML.bak_20260615_221045
```

## Технічні деталі `read_tag` / `write_tag`

Regex для читання:

```python
rf"<{tag}>\s*(.*?)\s*</{tag}>"
```

Прапорець `re.S` (DOTALL) — на випадок багаторядкових значень.

**Обмеження:** `write_tag` змінює **перше** входження тега. Для повторюваних тегів у фракціях — `replace_all_tags`.

---

## Що редактор НЕ вміє (навмисно)

- Редагувати файли всередині `Config.meg`
- Редагувати окремі `UNITS_*.XML` (немає UI)
- Валідувати XML-схему Petroglyph (гра сама повідомить про помилку)
- Застосовувати зміни без перезапуску гри

---

## Типові помилки

| Помилка | Причина | Рішення |
|---------|---------|---------|
| «Тег не знайдено» | опечатка або інша версія XML | перевірити файл вручну |
| Гра не стартує | зламаний XML | відновити `.bak_*` |
| ШІ теж «читерить» | глобальні теги | використати Trainer замість XML HP |
