# 02. Структура гри Empire at War — Forces of Corruption

## Розташування установки

Колекція **Star Wars Empire At War Collection** містить дві незалежні установки:

| Папка | EXE | Гра |
|-------|-----|-----|
| `data1\` | `sweaw.exe` | Empire at War (базова) |
| `data2\` | `swfoc.exe` | Forces of Corruption (доповнення) |

Усі подальші дії стосуються **`data2`** — FOC.

---

## Файл `MegaFiles.xml` — «маніфест» архівів

Шлях: `data2\Data\MegaFiles.xml`

```xml
<Mega_Files>
    <File> Data\Config.meg </File>
    <File> Data\Maps.meg </File>
    <File> Data\Models.meg </File>
    ...
</Mega_Files>
```

**Як це працює:**

1. При старті гра читає `MegaFiles.xml`.
2. Завантажує перелічені `.meg` файли **по черзі**.
3. Будує **єдину віртуальну таблицю файлів** (Master File Table).
4. Якщо один і той саме ім'я файлу є в кількох MEG — **перемагає останній** у списку.

**Важливий виняток (моддинг):**

Файли, фізично лежачі в `Data\XML\` на диску, мають **вищий пріоритет**, ніж однойменні файли всередині `Config.meg`. Саме тому достатньо покласти змінений `GAMECONSTANTS.XML` у папку — гра його підхопить без розпаковки всього архіву.

---

## Основні MEG-архіви в `data2`

| Файл | Приблизний розмір | Вміст |
|------|-------------------|-------|
| `Config.meg` | ~18 МБ | XML, Lua, AI-скрипти, константи |
| `Maps.meg` | великий | карти `.ted` |
| `Models.meg` | великий | 3D-моделі `.alo` |
| `Textures.meg` | великий | текстури |
| `RussianSpeech.meg` | середній | озвучення (RU) |
| `Patch.meg` | ~33 МБ | патч-дані (моделі, текстури) |

На момент дослідження у `data2\Data\XML\` було лише **7 файлів** — часткова розпаковка або ручне копіювання моддером.

## Виконуваний файл `swfoc.exe`

### Архітектура

Перевірка PE-заголовка показала:

```
Machine: 0x14C → 32-bit (x86)
Розмір: ~8.97 МБ
```

**Наслідки для трейнера:**

- Процес гри — **32-бітний**.
- Трейнер (64-біт Python + pymem) на Windows **може** читати 32-бітну пам'ять через WOW64.
- Вказівники в пам'яті — **4 байти** (uint32).
- `float` — IEEE 754, 4 байти, little-endian.

### Корисні рядки всередині EXE

Під час аналізу бінарника `strings` знайдено:

| Рядок | Кількість | Значення для моддингу |
|-------|-----------|----------------------|
| `Take_Damage` | 7 | система урону |
| `Get_Credits` | 1 | економіка |
| `Player` | 88 | ідентифікація гравців |
| `Ability_Cooldown_Time_In_Secs` | 1 | перезарядка здібностей |
| `Build_Cost` | 5 | вартість будівництва |
| `Unit_Cap` | 3 | ліміт юнітів |
| `Credits:%f` | — | кредити як **float** |

Це підтвердило: для сканера кредитів потрібно шукати і `int`, і `float`.

## Ієрархія ігрових даних (XML)

Типова структура всередині `Config.meg`:

```
Data\XML\
├── GAMECONSTANTS.XML      ← глобальні константи рушія
├── PLANETS.XML            ← планети, дохід, позиції
├── FACTIONS.XML           ← Імперія, Повстанці
├── EXPANSION_FACTIONS.XML ← Underworld, Zann Consortium (FOC)
├── UNITS_*.XML            ← характеристики юнітів
├── STARBASES.XML          ← космічні станції
├── HEROES_*.XML           ← герої
├── CAMPAIGNS_*.XML        ← кампанії
└── ...
```

**1046** записів у таблиці імен `Config.meg`, з них **~628** — XML-подібні файли.

## Режими гри (де застосовуються моди)

| Режим | XML-константи | Трейнер |
|-------|---------------|---------|
| Galactic Conquest (галактика) | `Credit_Cap_Per_Planet`, `PLANETS.XML` | кредити, ліміт юнітів, будівництво |
| Tactical Space (космічний бій) | `Object_Max_Health_Multiplier_Space` | HP, тактичні кредити |
| Tactical Land (наземний бій) | `Object_Max_Health_Multiplier_Land` | HP, здібності |
| Skirmish | `Min/Max_Skirmish_Credits`, `FACTIONS.XML` | тактичні кредити, pop cap |
| Campaign | сюжетні скрипти Lua + XML | герої можуть «вмерти назавжди» через скрипт |

## Консольні чити (вбудовані в гру)

Окремо від наших інструментів, FOC має консоль (`~`):

| Команда | Ефект |
|---------|-------|
| `cash` | +1 000 000 кредитів |
| `instant_action` | миттєве будівництво |
| `noclip` | вільна камера |
| `turbo` | прискорення |

**Обмеження:** консоль не дає тонкого «тільки для гравця» і не завжди доступна в кампанії.

---

## Чому обрали `data2`, а не `data1`

Користувач явно назвав **Forces of Corruption**. FOC додає:

- фракцію **Underworld** (Tyber Zann);
- `EXPANSION_FACTIONS.XML`;
- окремі юніти в `UNITS_HERO_UNDERWORLD_*.XML`;
- окремий `swfoc.exe`.

Трейнер шукає процеси `swfoc.exe` і `sweaw.exe`, але основна конфігурація орієнтована на FOC.

## Діаграма завантаження даних гри

```
swfoc.exe
    │
    ├─► MegaFiles.xml
    │       │
    │       ├─► Config.meg ──► XML, Lua (віртуальна ФС)
    │       ├─► Maps.meg
    │       └─► ...
    │
    └─► Data\XML\*.XML  ──► ПЕРЕКРИВАЄ Config.meg (якщо файл існує на диску)
```
