# 14. Гарячі клавіші трейнера

## Призначення

Система гарячих клавіш дозволяє виконувати будь-яку дію трейнера **одним натисканням клавіші**, не перемикаючись у вікно трейнера. Клавіші працюють **глобально** — навіть коли `swfoc.exe` у фокусі.

---

## Технічна реалізація

| Компонент | Файл |
|-----------|------|
| Менеджер hotkey | `trainer/hotkeys.py` → клас `HotkeyManager` |
| Збереження | `trainer/hotkey_store.py` → `trainer_hotkeys.json` |
| UI | `trainer/app.py` |

### Бібліотека `keyboard`

```
pip install keyboard>=0.13.5
```

Використовує низькорівневий хук Windows (`SetWindowsHookEx`). Потребує **прав адміністратора** для надійної роботи в грі.

### Реєстрація hotkey

```python
keyboard.add_hotkey(
    "ctrl+f5",
    callback,
    suppress=False,           # не блокує клавішу в грі
    trigger_on_release=False, # спрацьовує при натисканні
)
```

При закритті трейнера або призначенні нової клавіші — `keyboard.remove_hotkey(handle)`.

## Як призначити клавішу

### Спосіб A — кнопка ⌨ біля дії

1. Знайдіть потрібний рядок (значення або чит)
2. Натисніть **⌨**
3. У діалозі натисніть клавішу (можна з модифікаторами)
4. **Esc** — скасувати

### Спосіб B — блок «Гарячі клавіші»

1. Оберіть дію у випадаючому списку
2. **Назначити клавішу**
3. Натисніть клавішу в діалозі

### Відображення

Призначена клавіша показується **жовтим** (`#ffd700`) біля дії, наприклад: `Ctrl + F5`.

Також з'являється у списку внизу:

```
F5  →  Застосувати — Галактичні кредити
```

## Підтримувані клавіші та модифікатори

### Модифікатори

- `Ctrl`
- `Alt`
- `Shift`
- `Win` (windows)

Комбінації записуються як `ctrl+shift+f5` (внутрішній формат бібліотеки `keyboard`).

### Спеціальні клавіші (відображення в UI)

| Внутрішнє ім'я | Відображення |
|----------------|--------------|
| `space` | Пробіл |
| `enter` | Enter |
| `esc` | Esc |
| `f1`…`f12` | F1…F12 |
| `up` / `down` / `left` / `right` | ↑ ↓ ← → |
| `page up` / `page down` | PgUp / PgDn |

### Обмеження

| Правило | Деталі |
|---------|--------|
| Одна клавіша — одна дія | Нова прив'язка замінює стару для тієї ж клавіші |
| Одна дія — одна клавіша | Повторне призначення замінює попередню клавішу для дії |
| Конфлікт | Призначення тієї ж клавіші іншій дії — попередня дія втрачає клавішу |

```json
  ]
}
```

Файл у корені `EAW_FOC_Tools\`. Завантажується при старті трейнера.

## Потік виконання при натисканні

```
[Клавіша в грі]
       │
       ▼
keyboard hook (фоновий потік)
       │
       ▼
HotkeyManager._dispatch(action_id)
       │
       ▼
app._on_hotkey_pressed → self.after(0, ...)  ← GUI thread
       │
       ▼
app._execute_hotkey_action(action_id)
       │
       ├─ apply:*  → _apply_one(silent=True)
       ├─ pull:*   → _pull_one(silent=True)
       ├─ toggle:* → інверсія BooleanVar + _toggle_cheat
       └─ pull_all → _pull_all_from_game(silent=True)
```

**`silent=True`:** при помилці (немає підключення, некоректне число) — запис у журнал, **без** popup messagebox (щоб не переривати гру).

## Захоплення клавіші (capture)

Під час призначення:

1. `HotkeyManager.pause()` — тимчасово знімає всі hotkey
2. `keyboard.hook()` з `suppress=True` — перехоплює натискання
3. `_event_to_hotkey()` — збирає модифікатори + основну клавішу
4. `Esc` → скасування (`None`)
5. Таймаут 30 секунд
6. Після завершення — `resume()` / `_reload_hotkeys()`

## Усунення проблем

| Проблема | Рішення |
|----------|---------|
| Клавіша не працює в грі | Запуск трейнера **від адміна** |
| Клавіша працює лише в трейнері | Те саме — права адміна для `keyboard` |
| Після призначення нічого не відбувається | Перевірте підключення до гри |
| F-клавіші конфліктують з грою | Використайте `Ctrl+F…` |
| «Не вдалося призначити» у журналі | Невалідна комбінація для `keyboard` — спробуйте іншу |
