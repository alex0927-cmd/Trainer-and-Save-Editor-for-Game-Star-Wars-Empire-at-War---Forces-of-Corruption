# 15. Файлове логування трейнера

## Призначення

Опційний режим, який записує **повний журнал** усіх подій трейнера у текстовий файл на диску. Корисно для:

- діагностики проблем з калібруванням
- звітування про баги (прикріпити `trainer.log`)
- відстеження послідовності дій під час сесії

---

## Увімкнення

У блоці **«Журнал»** внизу вікна трейнера:

```
☐ Записувати все у файл    (EAW_FOC_Tools\trainer.log)
```

| Параметр | Значення |
|----------|----------|
| За замовчуванням | **ВИМКНЕНО** |
| Файл | `trainer.log` |
| Розташування | Поруч із трейнером (див. нижче) |
| Кодування | UTF-8 |
| Режим | Append (дописування, не перезапис) |

---

## Шлях до файлу

Визначається функцією `trainer_root()` у `trainer/file_logger.py`:

| Режим запуску | Шлях |
|---------------|------|
| `Запуск_Трейнера.bat` / `python trainer_main.py` | `EAW_FOC_Tools\trainer.log` |
| `EAW_FOC_Trainer.exe` | `dist\trainer.log` (папка з exe) |

Повний приклад:

```
C:\Games\Star Wars Empire At War Collection\EAW_FOC_Tools\trainer.log
```

## Що записується

### Завжди (коли галочка увімкнена)

Усе, що потрапляє в журнал GUI через `_log()`:

- підключення / відключення від `swfoc.exe`
- повідомлення `PlayerTrainer.log()` (калібрування, чити, помилки)
- результати авто-зв'язування
- призначення / видалення гарячих клавіш

### Додатково через `_log_event()`

| Подія | Приклад рядка |
|-------|---------------|
| Старт / закриття | `Трейнер запущено` |
| Підключення | `Підключення: swfoc.exe PID=12345` |
| Застосувати значення | `Застосувати [credits] = 9999999.0` |
| Перемкнути чит | `Чит [god_mode]: УВІМК` |
| Гаряча клавіша | `Гаряча клавіша → Застосувати — Галактичні кредити` |
| Авто-зв'язати | `Авто-зв'язати [credits] — крок 1` |
| Увімк/вимк файлового логу | `Запис у файл увімкнено → …` |

### Live-значення (категорія `LIVE`)

Коли увімкнено файлове логування, при **зміні** «Поточного» значення з гри:

```
[14:32:01.234] [LIVE] Live [credits]: None → 5000
[14:35:10.891] [LIVE] Live [credits]: 5000 → 4500
```

Запис лише при зміні — не кожну секунду однакове число.

### Помилки (категорія `ERROR`)

При помилці підключення — повний Python traceback:

```
[14:00:00.123] [ERROR] Traceback (most recent call last):
  ...
```

## Формат рядка

```
[ЧАС] [КАТЕГОРІЯ] Текст повідомлення
```

| Категорія | Джерело |
|-----------|---------|
| `LOG` | Загальні повідомлення журналу (за замовчуванням) |
| `EVENT` | Дії користувача та системні події |
| `LIVE` | Зміна live-значень з RAM |
| `ERROR` | Traceback винятків |

Час: `HH:MM:SS.mmm` (мілісекунди).

### Межі сесії у файлі

При увімкненні:

```
========================================================================
Сесія розпочата: 2026-06-15 19:30:00
Файл: C:\Games\...\EAW_FOC_Tools\trainer.log
========================================================================
```

При вимкненні галочки або закритті трейнера:

```
Сесія завершена: 2026-06-15 20:15:00
========================================================================
```

## Технічна реалізація

### Клас `SessionFileLogger`

Файл: `trainer/file_logger.py`

```python
class SessionFileLogger:
    def set_enabled(on: bool) -> Path   # відкрити/закрити файл
    def write(message, category="LOG")  # потокобезпечний запис
    def close()                         # завершити сесію
```

### Потокобезпечність

- `threading.Lock` — запис з GUI-потоку та з `_maintenance_loop` (через `self.log` callback) не конфліктує
- `flush()` після кожного рядка — мінімальна втрата даних при аварійному закритті

### Інтеграція в `app.py`

```python
def _log(self, msg: str, category: str = "LOG") -> None:
    # 1. Показати в ScrolledText
    # 2. self._file_logger.write(msg, category=category)

def _on_close(self) -> None:
    self._file_logger.close()  # гарантоване закриття файлу
```

**Важливо:** події **до** увімкнення галочки у файл **не** записуються. Після увімкнення — лише нові події.

## Рекомендації

| Ситуація | Дія |
|----------|-----|
| Діагностика «не працюють кредити» | Увімкнути лог → авто-зв'язати → застосувати → надіслати `trainer.log` |
| Довга сесія | Файл росте; періодично архівуйте або видаляйте старий `trainer.log` |
| Конфіденційність | Лог містить PID процесу, адреси RAM — не публікуйте без потреби |
