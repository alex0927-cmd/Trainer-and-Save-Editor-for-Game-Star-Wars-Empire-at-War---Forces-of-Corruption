# 06. Проблема «тільки для гравця»: чому XML недостатній

## Вимога користувача

> Повинно додавати це усе **тільки мені**, а **противника** щоб **не чіпало**.

Це фундаментально змінило архітектуру рішення.

---

## Чому XML-моди зачіпають ШІ

### Приклад 1: множник здоров'я

```xml
<Object_Max_Health_Multiplier_Land> 99999.0 </Object_Max_Health_Multiplier_Land>
```

Рушій застосовує множник при створенні **будь-якого** `GameObjectType` у наземному режимі. Немає тега «тільки для Player_0».

### Приклад 2: дохід планет

```xml
<Planet_Credit_Value>99999</Planet_Credit_Value>
```

Планета дає однаковий дохід **власнику** — якщо ШІ захопить планету, він теж отримає 99999.

### Приклад 3: фракційні множники

`FACTIONS.XML` описує **шаблон фракції** Rebel/Empire. Якщо ви граєте за Імперію з `Credits_Accumulation_Factor=100`, а ШІ — за Повстанців, у них **теж** буде 100, бо їхня фракція описана в тому ж файлі з аналогічними змінами (ми змінили всі входження).

---

## Де в грі взагалі є поняття «гравець»

### У пам'яті під час гри

З `swfoc.exe` strings:

```
Player %d (%ls)
BroadcasterClass::Find_Player_From_ID
BroadcasterClass::Find_Player_From_Address
```

Кожен об'єкт (юніт, планета, флот) має **власника** (player slot / faction id). Саме це використовують Cheat Engine таблиці для player-only патчів.

### У XML

Є `Affiliation`, `Owner`, `Faction` на рівні юнітів і планет, але **глобальні константи** в `GAMECONSTANTS.XML` не мають прапорця «локальний людський гравець».

---

## Порівняння підходів

| Критерій | XML-мод | Трейнер (пам'ять) |
|----------|---------|-------------------|
| Тільки гравець | ❌ | ✅ (з калібруванням / хуками) |
| Стабільність після патчу гри | ❌ (може зламатися) | ⚠️ (AoB може змінитися) |
| Потрібен адмін | Ні | Так |
| Змінює файли | Так | Ні |
| Працює без гри запущеної | Так (після рестарту) | Ні |
| Антивірус | Спокійно | Може тривожити |

---

## Як спільнота вирішує player-only

За даними **FearLess Cheat Engine** / **Open Cheat Tables** для FOC 64-bit Steam:

| Чит | Player-only? |
|-----|--------------|
| Infinite Credits | Так (окремий гаманець гравця) |
| God mode v1.04 | Так (перевірка власника юніта) |
| 1 sec / 1 credit build (Conquest) | Заявлено player-only у v1.02 |
| 1 sec / 1 credit build (Skirmish) | Іноді зачіпає ШІ (скарги на форумі) |
| Max unit cap 99k GC | Spoof лічильника для гравця |

Ми повторили **ту саму ідею**: калібрування + патчі з перевіркою контексту.

## Архітектурне рішення проєкту

```
┌─────────────────────────────────────────────────────────┐
│  EAW_FOC_Editor.exe                                     │
│  Глобальні зміни XML (баланс, тестування, пресети)      │
└─────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────┐
│  EAW_FOC_Trainer.exe                                    │
│  Runtime: кредити, HP, ліміти — націлено на гравця      │
└─────────────────────────────────────────────────────────┘
```

Користувачу рекомендовано:

- **Не використовувати** XML-пресет з HP×99999, якщо хоче чесного ШІ — замість цього **трейнер God Mode**.
- XML залишити для економіки в single-player, якщо комфортно.
