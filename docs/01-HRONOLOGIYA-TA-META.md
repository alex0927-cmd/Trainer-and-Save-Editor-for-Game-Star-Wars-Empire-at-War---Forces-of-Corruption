# 01. Хронологія, мета та вихідні вимоги

## Початковий запит користувача (етап 1)

Користувач попросив для **Star Wars: Empire at War — Forces of Corruption**:

1. **Безкінечні життя** для юнітів.
2. **Безкінечні гроші** (кредити).

Робоча папка в IDE вказувала на:

```
C:\Games\Star Wars Empire At War Collection\data2
```

Це каталог доповнення **Forces of Corruption** (FOC), а не базової гри Empire at War (`data1`).

---

## Початковий запит користувача (етап 2)

Після XML-модів користувач уточнив:

1. Створити **редактор гри** з **графічним інтерфейсом**.
2. Створити **трейнер +10** з функціями:
   - безсмертя;
   - безлімітні ресурси;
   - безлімітні кошти;
   - навички без перезарядки;
   - нескінченна кількість юнітів;
   - та інші бонуси.
3. Усе має бути **скомпільовано в `.exe`** для Windows.
4. **Критично:** чити мають діяти **тільки на гравця**, противник (ШІ) **не повинен** отримувати ті самі бонуси.

Після базового трейнера користувач додав вимоги:

## Чому виникло два рішення, а не одне

Під час першого етапу було зрозуміло:

- У грі Petroglyph/LucasArts дані зберігаються в **MEG-архівах** (`Config.meg` тощо).
- Моддинг через **розпаковані XML** у `Data\XML\` — стандартна практика спільноти.
- Більшість тегів у `GAMECONSTANTS.XML`, `PLANETS.XML`, `FACTIONS.XML` — **глобальні константи**.

Тобто зміна, наприклад:

```xml
<Object_Max_Health_Multiplier_Land> 99999.0 </Object_Max_Health_Multiplier_Land>
```

підвищує здоров'я **усіх** юнітів у наземних боях — і ваших, і ворожих.

Коли з'явилась вимога «не чіпати противника», XML-моддинг **не міг** стати єдиним рішенням. Потрібен був **трейнер**, що працює з пам'яттю процесу та/або вставляє перевірку «це локальний гравець?».

## Логічний ланцюжок рішень

```
Запит: гроші + безсмертя
        ↓
Де зберігаються параметри? → Config.meg / Data\XML\
        ↓
Що вже розпаковано в data2? → GAMECONSTANTS.XML, кілька інших
        ↓
Які теги відповідають за HP і кредити? → grep по XML
        ↓
Змінити теги → миттєвий ефект після перезапуску гри
        ↓
Новий запит: GUI + exe + тільки для гравця
        ↓
XML = глобально → потрібен runtime trainer
        ↓
swfoc.exe = 32-bit → pymem + сканер + AoB-патчі
        ↓
EAW_FOC_Tools: Editor.exe + Trainer.exe
```

## Принципи, яких дотримувались при розробці

1. **Мінімальний diff** — не переписувати всю гру, лише потрібні теги/модулі.
2. **Розпаковані XML мають пріоритет** — не чіпати `Config.meg` напряму (ризик зламати архів).
3. **Резервні копії** — редактор робить `.bak_YYYYMMDD_HHMMSS` перед записом.
4. **Розділення відповідальності** — редактор = файли; трейнер = пам'ять.
5. **Чесність щодо обмежень** — XML не може бути player-only; трейнер потребує калібрування.
